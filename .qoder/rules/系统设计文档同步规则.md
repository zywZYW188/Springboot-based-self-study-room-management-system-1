# 系统设计文档同步规则

## 规则定义

本规则约定：所有基于代码生成的**系统设计文档**、**数据库设计文档**及相关技术文档，必须统一放到 `.qoder/repowiki/` 目录下，并与代码变更保持同步更新。

## 适用范围

- 系统架构设计文档
- 数据库表结构设计文档（含建表语句 DDL）
- 数据模型设计文档
- API 接口设计文档
- 核心功能模块设计文档
- 配置与部署说明文档

## 目录规范

| 文档类型 | 存放路径 |
|---------|---------|
| 系统架构设计 | `.qoder/repowiki/zh/content/系统架构设计.md` |
| 数据模型设计 | `.qoder/repowiki/zh/content/数据模型设计/` |
| 数据库建表语句 | `.qoder/repowiki/zh/content/数据模型设计/数据库建表语句.md` |
| API 接口文档 | `.qoder/repowiki/zh/content/API接口文档/` |
| 核心功能模块 | `.qoder/repowiki/zh/content/核心功能模块/` |
| 前端页面设计 | `.qoder/repowiki/zh/content/前端页面设计/` |
| 工具类库说明 | `.qoder/repowiki/zh/content/工具类库/` |
| 项目概述 | `.qoder/repowiki/zh/content/项目概述.md` |
| 快速开始 | `.qoder/repowiki/zh/content/快速开始.md` |
| 技术栈详解 | `.qoder/repowiki/zh/content/技术栈详解.md` |
| 配置与部署 | `.qoder/repowiki/zh/content/配置与部署.md` |

## 同步要求

1. **新增文档**：首次生成文档时，直接写入 `.qoder/repowiki/zh/content/` 下对应的目录或文件
2. **更新文档**：代码变更导致设计变化时，必须同步更新 repowiki 下对应文档
3. **禁止位置**：不允许仅在项目根目录、`wiki/` 目录或其他位置生成设计文档而不同步到 repowiki
4. **引用规范**：文档中引用的代码文件路径使用 `file://` 格式，指向项目实际文件路径
